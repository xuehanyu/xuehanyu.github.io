JS基础——语法

2018-07-22  

语法概述

下面是值的类型

五种基本类型

  • 字符串
    ‘hello’
    “hello”
  • 数字
    1.234
    1234
  • 布尔值
    true
    false
  • null
  • undefined

复杂数据类型-Object

下面是几个基础语法的例子

var x     //  声明一个变量x 
x = 3 + z   //  给变量x赋值
foo(a,b)   //  函数foo调用并传入形参a,b
obj.baz(3)   //  对象的方法调用
// 条件语句
if（x===2）{  // 判断x与2是否相等
  x=234  
}
//  函数生命 , 变量为a,b
function(a,b){
  return a+b
}

表达式与语句

表达式

表达式将会产生一个值，它可以出现在任何需要值的地方，如
x + 10
foo（2，4）

语句

语句表示一种行为，如if语句、循环语句，一个程序基本上就是一个语句的序列。

注：凡是js期望语句的地方都可以写表达式，这样的语句叫做表达式语句。反之则不然，不能在需要表达式的地方适用语句，如判断语句不能成为函数的参数

条件语句与条件表达式（某些情况下可以互通）

//  使用条件语句
  if(score>60){
    str = '及格'
  } else {
    str = '不及格'
  }
//  如用条件表达式如下 （条件运算符或三元表达式）
  str = score > 60 ? '及格' : '不及格'

严格模式

ES5 提供了严格模式，在严格模式中，JavaScript代码会更加简洁，会有更少不安全的特性，更多的警告和更加合理的代码。普通模式（非严格模式），有时也被成为“宽松模式”。

启用严格模式

可以在JavaScript文件或者script标签中的第一行加入以下代码来启动严格模式
'use strict'
或者可以在函数作用域中启用严格模式
function foo(){ 'use strict' ... }

严格模式中变量必须背声明

在严格模式中，所有变量必须被显示声明，在宽松模式中，未显示声明的变量将会创建一个全局变量

// 宽松模式
  function foo(){
    x = 111
  }
  foo()   //  创造了一个全局变量x
  console.log(x)   //  111;
//  严格模式
function foo(){
 'use strict'
  x = 111
}
foo()   //  抛出异常， ReferenceError: x is not defined

函数必须在作用域的顶部声明

在严格模式中，所有的函数必须在顶级作用域声明，这意味着你不能在一个块内声明函数，如果这样做，将会抛出描述性错误SyntaxError；

  function foo(){
    'use strict'
    {
      // SyntaxError；
      function bar(){
      }
    }
  }

  function foo(){
    'use strict'
    {
      //  OK
      var bar = function (){
      }
    }
  }

无方法的函数中 this的值为undefined

// 在宽松模式中
  function foo(){
    console.log(this===window)   // true
  }
//  在严格模式中
function foo(){
  'use strict'
  console.log(this===undefined)   // true
}

在严格模式中，设置或删除不可改变的属性会抛出异常

严格模式中，对属性的非法操作会抛出异常。如，对只读属性的设定操作会抛出一场，同样不可以删除不可配置的属性，如：

var str = "hello"
function foo() {
  str.length = 8  //  静默失败
  console.log(str.length)  //  5
}

function foo() {
  'use strict'
  str.length = 8  //  抛出异常 TypeError: Cannot assign to read only property 'length' of string 'hello'
}

在严格模式中，不合格标识符不能删除

在宽松模式中，可以用下面的方式删除全局变量foo
delete foo
在严格模式中，想要删除不合格标识符时将会抛出异常，但是可以通过以下方式进行删除全局变量

delete window.foo //  浏览器
delete global.foo  //  Node.js
delete this.foo  //  全局作用域
